App vibe
Fun · Social · Celebratory
Clean iOS-native feel. Not hardcore-athlete, not Peloton.
Core concept: Every wodi entry is a win worth celebrating. The dark canvas makes results pop. Yellow = energy, reward, PR. Stickers are the primary delight mechanic — tactile, physical, shareable. The app should feel like peeling a gold star off a sheet and sticking it on your logbook.

05 — Page Layout & Scroll Model

08 — Do / Don't
✓ Do
Use yellow for PRs, stickers, hero numbers, and primary CTA only
Use Barlow Condensed 900 for all numbers and WOD names
Rotate stickers ±2–6° for physical/tactile feel
Keep screen edge padding at 20px consistently
UPPERCASE + wide letter-spacing for all eyebrow labels
Show difficulty chip only on metcons (AMRAP, For Time, EMOM)
Display difficulty as "{number} · {label}" format
Keep stats card fixed — never inside the swipe pager
Give each exercise page its own title and sticker(s)
Use haptics on every weight increment and PR moment
✗ Don't
Don't show difficulty badge on strength/lifting WODs
Don't use yellow for body text or secondary labels
Don't mix cyan/blue accents — one accent color only
Don't use Inter, Roboto, system-ui — Barlow only
Don't add gradients to text
Don't use light backgrounds in the main app flow
Don't add more than 3 stickers per exercise page
Don't put the stats card inside the swipe region
Don't use border-radius below 6px on interactive elements
Don't use emoji anywhere in the UI
